# Contributing

REHT Standard is a public specification, not a production-code repository.

Contributions must be submitted through pull requests and include:

- problem statement;
- proposed normative or non-normative change;
- compatibility impact;
- security impact;
- affected schemas and examples;
- migration notes where applicable.

Normative language uses MUST, MUST NOT, SHOULD, SHOULD NOT and MAY as defined by RFC 2119 and RFC 8174.

Do not submit private VALO source code, customer information, credentials, unpublished patent material, internal thresholds or proprietary evaluation logic.

By submitting a contribution, you agree that it may be licensed under Apache License 2.0 and that you have the right to submit it.
