# Security Policy

Do not disclose suspected vulnerabilities through public issues.

Report security concerns privately to the repository owner through GitHub Security Advisories when available, or through an established private contact channel.

Include:

- affected specification version;
- affected object or schema;
- impact;
- reproduction or attack sequence;
- suggested mitigation;
- whether production systems may be affected.

This repository contains no production runtime. Reports concerning private VALO implementations must not include confidential implementation details in this public repository.
